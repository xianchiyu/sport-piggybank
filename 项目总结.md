# 项目总结

## 任务目标
安卓 App"运动存钱罐"。运动和健康饮食行为转化为虚拟硬币存入储钱罐，作为非必需品消费的唯一资金来源。违规双轨惩罚（现金投ETF + 积分扣回）。季度末余额提现到余额宝。

## 技术方案变更
- ~~微信小程序 + 云开发~~ → **安卓 WebView 壳 + 纯前端**
- 原因：免去小程序注册流程；本地通知更可靠；数据完全在本地
- CI 编译：push 到 GitHub → Actions 自动编译 → 下载 APK

## 关键决策记录
- 币值锚定真实金额：铜=1毛，银=1块，金=10块
- 连续加成用倍率（×1.0→1.2→1.5→1.8），各任务独立
- 跑步/走路二选一，共用运动连续天数
- 惩罚双轨：现金递进（三角数，15天周期）+ 积分连带扣回
- 季度清零提现，手动转账到余额宝
- 社交豁免：仅限晚餐，当天使用，免罚金且保留连续天数，每月3次（2026-07-22 确认）
- MVP 暂不做自定义任务，先跑通核心流程
- 纯手动打卡，不可补签
- 技术栈从微信小程序改为安卓 WebView 壳（2026-07-16）
- 数据用 SharedPreferences（MVP），后续可升级 SQLite
- CI 编译交给 GitHub Actions，本地零工具链
